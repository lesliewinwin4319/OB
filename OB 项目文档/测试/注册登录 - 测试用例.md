## 1. Test Strategy Summary

测试重点在于**微信授权与 UID 唯一性**、**账号状态机（PENDING_PROFILE / ACTIVE）驱动的导航恢复**、**资料填写页的输入边界与头像算法**，以及**好友引导页的返回禁止**四条主干路径的正确性与鲁棒性。

## 3. Potential Risks

1. **昵称长度已统一为 20 字符**：PRD、客户端方案、服务端方案均对齐为 maxLength=20，前后端一致。

2. **JWT status 字段与 DB 不一致**：服务端方案已指出此风险（风险3），Guard 须以 DB 实时状态为准。测试时需验证完善资料后使用旧 Token 是否仍被正确放行。

3. **侧滑手势禁用通过 UINavigationController 扩展实现**：属于系统私有行为，iOS 大版本升级可能失效；测试需覆盖目标最低支持系统版本。

4. **Emoji 截断边界**：客户端使用 `String.prefix(30)` 处理，理论安全；但粘贴超长 Emoji 组合字符串（如多个家庭 Emoji）时，需人工验证截断后显示不乱码。

5. **微信头像 URL 时效性**：微信原始 URL 约 3 天有效期，CDN 异步任务本期不上线，若任务迟迟未消费，新用户头像将在 3 天后失效。测试阶段需确认降级逻辑可用。

6. **资料填写页键盘聚焦时机**：客户端方案使用 `asyncAfter(0.1s)` 对齐页面动画，弱设备上页面动画更慢，可能出现键盘弹起早于页面过渡完成的视觉问题。

7. **WebView Sheet 加载失败**：当前 URL 为百度，网络异常时 WebView 会加载失败，需确认是否有错误页兜底。

8. **好友引导页 onDisappear 恢复手势**：客户端方案中 `onDisappear` 会重新启用侧滑手势，理论上不会触发（因为进入首页时整个 NavigationStack 已替换），但需验证不同跳转路径下首页的侧滑手势是否正常。

## 4. Blocker 定义

以下情况属于阻塞级（P0），任意一项未通过则**不允许上线**：

| 编号 | Blocker 场景 |
|------|-------------|
| B-01 | 新用户完成微信授权后无法跳转资料填写页（注册主流程断裂） |
| B-02 | 同一微信账号重复注册，产生两个 UID（数据一致性破坏） |
| B-03 | ACTIVE 老用户登录后未直接进首页，而是进入资料填写页（用户体验严重回退） |
| B-04 | PENDING_PROFILE 用户强退后重启，无法恢复至资料填写页（注册流程卡死） |
| B-05 | 输入框为空时「下一步」仍可点击并触发提交（产品逻辑违反） |
| B-06 | 昵称超出 30 字符未被截断，且提交成功写入数据库（数据污染） |
| B-07 | 好友引导页侧滑/返回键可返回资料填写页（产品明确禁止的导航行为） |
| B-08 | Token 失效后 App 不重定向登录页，导致用户卡在登录态中间态 |
| B-09 | 昵称长度前后端限制不一致且未修复，导致用户无法完成注册（TC-049 确认为 Blocker）|
| B-10 | 未安装微信时点击登录按钮导致 App 崩溃 |

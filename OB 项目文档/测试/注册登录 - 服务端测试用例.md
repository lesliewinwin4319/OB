# 注册登录 - 服务端测试用例

版本：v1.0  
日期：2026-04-10  
负责人：Seth（服务端）  
测试范围：Auth 模块全链路（微信登录、资料完善、用户信息获取）

---

## 测试环境说明

- 当前阶段使用 Mock 微信登录：任意非空字符串均可作为 `code` 传入
- 需提前启动 PostgreSQL、Redis 服务
- 所有请求 Content-Type: application/json
- JWT Token 有效期：7 天

---

## 一、POST /auth/wechat/login — 微信登录 / 注册

### TC-AUTH-001 首次登录（新用户注册）

**前置条件：** 数据库中不存在该 openId 对应用户  
**请求：**
```json
{ "code": "test_code_001" }
```
**期望响应：** HTTP 201
```json
{
  "token": "<JWT>",
  "user": {
    "uid": "<snowflake_id>",
    "status": "PENDING_PROFILE",
    "nickname": null,
    "avatarUrl": null
  }
}
```
**验证点：**
- status 必须为 PENDING_PROFILE
- uid 为 64 位整型雪花 ID，字符串表示
- token 可解码，sub 字段等于 uid
- 数据库 users 表新增一条记录

---

### TC-AUTH-002 老用户再次登录（PENDING_PROFILE 状态）

**前置条件：** 用户已存在，status = PENDING_PROFILE  
**请求：**
```json
{ "code": "test_code_001" }
```
**期望响应：** HTTP 200
```json
{
  "token": "<JWT>",
  "user": {
    "uid": "<existing_uid>",
    "status": "PENDING_PROFILE",
    "nickname": null,
    "avatarUrl": null
  }
}
```
**验证点：**
- 数据库未新增记录（幂等）
- 返回的 uid 与已有记录一致

---

### TC-AUTH-003 老用户再次登录（ACTIVE 状态）

**前置条件：** 用户已存在，status = ACTIVE，nickname 已设置  
**请求：**
```json
{ "code": "test_code_active" }
```
**期望响应：** HTTP 200
```json
{
  "token": "<JWT>",
  "user": {
    "uid": "<uid>",
    "status": "ACTIVE",
    "nickname": "张三",
    "avatarUrl": "https://example.com/avatar.jpg"
  }
}
```
**验证点：**
- status 为 ACTIVE
- nickname、avatarUrl 正确返回

---

### TC-AUTH-004 code 为空字符串

**请求：**
```json
{ "code": "" }
```
**期望响应：** HTTP 400
```json
{
  "errorCode": "VALIDATION_ERROR",
  "message": "code is required"
}
```

---

### TC-AUTH-005 code 字段缺失

**请求：**
```json
{}
```
**期望响应：** HTTP 400，errorCode = VALIDATION_ERROR

---

### TC-AUTH-006 并发重复注册（相同 openId 同时发起两次请求）

**场景：** 模拟客户端抖动，同一 openId 在 50ms 内发起两次登录请求  
**期望结果：**
- 数据库仅创建一条用户记录（唯一索引保护）
- 两次请求均返回 200/201，uid 相同
- 不出现 500 错误或重复记录

## 二、POST /users/me/profile — 完善用户资料

### TC-PROFILE-001 正常完善资料

**前置条件：** 携带 PENDING_PROFILE 用户的有效 JWT  
**请求：**
```json
{
  "nickname": "张三",
  "avatarUrl": "https://cdn.example.com/avatar/abc.jpg"
}
```
**期望响应：** HTTP 200
```json
{
  "uid": "<uid>",
  "status": "ACTIVE",
  "nickname": "张三",
  "avatarUrl": "https://cdn.example.com/avatar/abc.jpg"
}
```
**验证点：**
- 数据库中 status 已变更为 ACTIVE
- nickname、avatarUrl 已持久化
- 再次调用 GET /users/me 返回 ACTIVE 状态

---

### TC-PROFILE-002 avatarUrl 为空（仅设置 nickname）

**请求：**
```json
{ "nickname": "李四" }
```
**期望响应：** HTTP 200，avatarUrl = null，status = ACTIVE  
**验证点：** avatarUrl 允许为 null，业务仍正常流转

---

### TC-PROFILE-003 nickname 为空字符串

**请求：**
```json
{ "nickname": "" }
```
**期望响应：** HTTP 400，errorCode = VALIDATION_ERROR

---

### TC-PROFILE-004 nickname 超长（> 20 字符）

**请求：**
```json
{ "nickname": "这个昵称超过了二十个字符的最大长度限制测试" }
```
**期望响应：** HTTP 400，errorCode = VALIDATION_ERROR

---

### TC-PROFILE-005 未携带 Token（未登录）

**请求：** 无 Authorization 头  
**期望响应：** HTTP 401
```json
{
  "errorCode": "UNAUTHORIZED",
  "message": "Missing or invalid token"
}
```

---

### TC-PROFILE-006 Token 伪造 / 篡改

**请求：** Authorization: Bearer invalid.token.here  
**期望响应：** HTTP 401，errorCode = UNAUTHORIZED

---

### TC-PROFILE-007 ACTIVE 用户重复提交资料完善

**前置条件：** 用户已是 ACTIVE 状态  
**请求：**
```json
{ "nickname": "新昵称" }
```
**期望响应：** HTTP 200（允许更新资料）  
**验证点：** nickname 更新成功，status 保持 ACTIVE，不回退状态

---

### TC-PROFILE-008 状态机越权：PENDING_PROFILE 用户访问其他受保护路由

**前置条件：** 用户 status = PENDING_PROFILE，持有有效 JWT  
**请求：** GET /some/protected/route  
**期望响应：** HTTP 403
```json
{
  "errorCode": "PROFILE_INCOMPLETE",
  "message": "Please complete your profile first"
}
```
**验证点：** 中间件拦截生效，不暴露任何业务数据

---

## 三、GET /users/me — 获取当前用户信息

### TC-ME-001 正常获取（ACTIVE 用户）

**前置条件：** 携带 ACTIVE 用户的有效 JWT  
**期望响应：** HTTP 200
```json
{
  "uid": "<uid>",
  "status": "ACTIVE",
  "nickname": "张三",
  "avatarUrl": "https://cdn.example.com/avatar/abc.jpg",
  "createdAt": "2026-04-10T07:00:00.000Z"
}
```

---

### TC-ME-002 PENDING_PROFILE 用户获取自己信息

**前置条件：** 用户 status = PENDING_PROFILE  
**期望响应：** HTTP 200（此路由豁免 PROFILE_INCOMPLETE 检查）
```json
{
  "uid": "<uid>",
  "status": "PENDING_PROFILE",
  "nickname": null,
  "avatarUrl": null
}
```
**验证点：** /users/me 在白名单内，不被 403 拦截

---

### TC-ME-003 Token 过期

**请求：** 使用已过期的 JWT  
**期望响应：** HTTP 401，errorCode = TOKEN_EXPIRED

---

### TC-ME-004 用户不存在（Token 合法但 uid 在数据库中已删除）

**场景：** 极端情况，Token 有效但对应用户记录被物理删除  
**期望响应：** HTTP 404，errorCode = USER_NOT_FOUND

## 四、安全与边界测试

### TC-SEC-001 SQL 注入尝试

**请求：**
```json
{ "code": "' OR 1=1 --" }
```
**期望结果：** 正常处理为普通字符串，不触发 SQL 错误，不暴露数据库信息

---

### TC-SEC-002 超大 Payload

**请求：** nickname 字段填入 10,000 字符  
**期望响应：** HTTP 400，VALIDATION_ERROR，不触发服务器崩溃

---

### TC-SEC-003 JWT Secret 轮换后旧 Token 失效

**场景：** 修改环境变量 JWT_SECRET 后重启服务  
**期望结果：** 旧 Token 验证失败，返回 401

---

### TC-SEC-004 并发 profile 提交（同一用户同时发起 5 个请求）

**场景：** 用户在网络抖动时同时发出多个资料完善请求  
**期望结果：** 所有请求均成功，数据库状态最终一致为 ACTIVE，无数据损坏

### Swift 6 编译报错修复

Xcode 26 + Swift 6 下遇到两轮报错，最终解法：

1. `AuthStateManager` 加 `@MainActor`，修复 `@Published` 并发警告
2. `ProfileSetupViewModel` 从 `ObservableObject + @Published` 迁移至 `@Observable` macro，彻底解决 `Color` 的 Sendable 问题
3. 所有 `onChange` 回调统一改为双参数签名 `{ _, newValue in }`

模拟器成功启动，流程跑通：登录页 → 资料填写页（含昵称预填、动态头像、字符限制 Toast）→ 好友引导页 → 首页。

## 遇到的问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Xcode 出现 Auth2/Utilities2 重复 group | 手动导入时误选 create groups 两次 | 删除原始 group（Remove Reference），保留新导入的 |
| LLDB RPC Server has exited 弹窗 | Xcode 26 beta 已知 debugger bug | 点 OK 忽略，不影响功能 |
| `hapticpatternlibrary.plist` 日志刷屏 | iOS 26 模拟器缺失文件，系统噪音 | 忽略，真机不出现 |
| 首次运行直接进资料填写页 | UserDefaults 残留上次状态 | 模拟器卸载 App 重装即可 |
| 昵称上限前后端不一致（30 vs 20） | PRD 原文与服务端方案不一致 | 统一为 20，PRD / 客户端方案 / 服务端方案 / 测试用例全部更新 |

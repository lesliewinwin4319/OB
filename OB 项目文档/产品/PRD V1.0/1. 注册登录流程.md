## 登录与注册流程技术规格文档

### 一、 登录页 (Login Screen)

- **法律合规性**：
    
    - 底部展示《用户协议》与《隐私政策》链接。
        
    - **交互**：点击从底部弹出 **WebView**（sheet 形式）。
        
    - **跳转逻辑**：当前 URL 缺省跳转至 `https://www.baidu.com`（待正式环境 URL 替换）。
        
- **微信授权逻辑**：
    
    - 调用 **WeChat SDK** 进行授权登录。
        
    - **数据处理**：获取 `userInfo`（头像 `headimgurl`、昵称 `nickname`）。
        
    - **异常处理**：
    - 若头像获取失败但昵称获取成功：昵称预填，头像按首字母生成规则自动生成。
    - 若头像和昵称均获取失败：清空预填，用户手动填写昵称，头像按输入内容自动生成。
    - 无微信头像不影响注册流程，用户可正常完成注册进入首页。
    - **登录失败**：弹出 Toast 提示「登录失败，请重试」，不做状态转移，按钮可再次点击。
        
- **用户生命周期 (UID)**：
    
    - **触发时机**：一旦微信授权流程完成并跳转至资料填写页，服务端即刻生成 **UID**，同时 status 立即设为 PENDING_PROFILE；资料填写完成提交后 status **立即**变为 ACTIVE，无需等到好友引导页。
        
    - **持久化逻辑**：
        - 若用户强退且**资料未完善（PENDING_PROFILE）**：再次授权后直接回到资料填写页。
        - 若用户强退且**资料已完善（ACTIVE）**：再次授权后直接进首页，不再经过资料填写页。
    
    - **一账号一 UID**：同一微信账号（OpenID）全局唯一对应一个 UID，不允许重复创建。

### 二、 个人资料填写页 (Profile Completion)

- **UI/UX 交互**：
    
    - **页面进入**：输入框自动获取焦点 (`autofocus`)，同步拉起系统软键盘。
        
    - **顶部文案**：`用真实的名字可以方便朋友找到你`。
        
    - **输入限制**：`maxLength = 20` 字符。达到上限后继续输入直接**截断丢弃**，同时弹出 **Toast** 提示。
        
    - **按钮状态**：输入框 `isEmpty` 时，“下一步”按钮置灰/禁用。
        
    - **键盘响应**：监听键盘右下角“确认/发送”键（`imeOptions: actionDone` 或 `returnKeyType: done`），触发逻辑等同于点击“下一步”。
        
- **动态头像生成算法 (Initials Avatar)**：
    
    若用户无微信头像，采用以下逻辑展示：
    
    - **字符提取逻辑**：
        
        1. **中文昵称**：提取首个汉字。
        2. **英文/拉丁昵称**：提取首字母并强制**大写**。
        3. **数字昵称**：展示首位数字。
        4. **Emoji 昵称**：展示首个 Emoji。
        5. **其他字符**（阿拉伯文、泰文、纯标点等）：展示 `?` 作为兜底占位符。
            
    - **视觉表现**：
        
        - **背景色**：进入页面时从预设 5-6 种**低饱和度莫兰迪色**中随机一次，固定不再变（`#B5C0D0`, `#CCD3CA`, `#F5E8DD`, `#EED3D9` 等）。
        - **字体颜色**：固定白色。
        - **样式**：字符居中。

### 三、 好友引导页 (Friend Discovery/Requests)

- **布局逻辑**：
    
    1. **第一部分：微信邀请**
        
        - UI 预留“微信加好友”入口按钮。
            
        - **当前逻辑**：点击触发 **Toast**（提示：功能开发中），后续补充跳转逻辑。
            
    2. **第二部分：好友申请列表**
        
        - **字段定义**：`Avatar` (头像), `Nickname` (昵称), `Action` (同意按钮)。
            
        - **空态**：展示占位文案，图片占位写「需要 LESLIE 提供占位图」。
            
- **导航约束**：
    
    - **下一步**：支持直接点击，跳转至 **App 首页**。
        
    - **物理返回键/侧滑手势**：**禁止返回**上一页（资料填写页）。

### 四、 开发建议 (Technical Tips)

- **头像组件**：建议封装一个 `AvatarComponent`，传入 `imageUrl` 和 `name`。内部逻辑：`if (!imageUrl) renderInitials(name)`。
    
- **状态管理**：建议在全局 State 中记录 `registrationStep`，确保用户非正常退出后能精准恢复现场。

- **昵称与头像的改查**：虽然本期不上线修改和搜索功能，服务端数据库设计和接口架构需预留支持，未来可直接扩展。

- **微信头像存储**：微信原始头像 URL 有时效性，服务端需异步下载并存至 CDN，以长期 URL 替换原始链接。

- **敏感词校验**：本期不做，后续根据运营需要补充。

- **Token 策略**：PENDING_PROFILE 状态与 ACTIVE 状态用户使用相同有效期的 JWT。

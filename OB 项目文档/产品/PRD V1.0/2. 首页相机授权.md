## 首页 - 相机与权限引导

**功能概述**：用户进入应用核心首页，通过后置摄像头完成“反自拍”第一步，同时处理关键的硬件访问授权逻辑。

### 1. 详细逻辑说明

- **触发条件/入口**：
    
    - **新用户**：在“添加好友”引导页点击【下一步】按钮，通过 `NavigationStack` 压栈或切换 `RootView` 进入。
        
    - **老用户**：App 启动冷启动后，若已完成注册引导，直接进入此页面。
        
- **权限状态机与业务流程**：
    
    1. **权限自检**：页面 `onAppear` 时，调用 `AVCaptureDevice.authorizationStatus(for: .video)`。
        
    2. **状态分发**：
        
        - **Case 1: `notDetermined` (首次)**：触发系统原生权限弹窗。
            
        - **Case 2: `authorized` (已授权)**：激活 `AVCaptureSession`，将后置摄像头预览流挂载至 `PreviewLayer`。
            
        - **Case 3: `denied / restricted` (已拒绝)**：保持黑屏，并弹出 **OB 自定义权限引导弹窗**（见 UI 规范）。
            
    3. **状态同步**：监听 `sceneDidBecomeActive` 通知。当用户从系统设置页返回 App 时，自动触发二次自检，实时刷新 UI 状态。
        
- **异常分支 (Fallback)**：
    
    - **无摄像头硬件**：展示纯黑屏
        
    - **启动失败**：若 `AVCaptureSession` 启动异常，Toast 提示“相机启动失败，请重启应用”。
        

### 2. UI/UX 规范

- **视觉反馈 (Morandi Color Palette)**：（简单做即可，后期以设计稿为准）
    
    - **背景色**：`#1A1A1A`（极深灰，非纯黑，保持莫兰迪质感）。
        
    - **弹窗色值**：背景使用 `#E0E0E0` (浅灰) 配合 `0.8` 的高斯模糊；按钮文字使用 `#333333`。
        
    - **拍照按钮**：外圈 `#D6D6D6`，内圈 `#FFFFFF`。禁用状态下 Alpha 设为 `0.3`。
        
- **交互细节**：
    
    - **权限弹窗**：文案：“OB 需要授权摄像头权限才能使用”。
        
        - **CTA 1**：【去设置】，点击执行 `UIApplication.shared.open(URL(string: UIApplication.openSettingsURLString)!)`。
            
        - **CTA 2**：【取消】，点击关闭弹窗，首页保持黑屏。
            
    - **点击聚焦**：支持原生 `Tap-to-Focus`。点击预览区域时，在点击处出现一个 **黄色方框（#E2C94C）** 逐渐缩小并消失，伴随轻微震动反馈 (`UIImpactFeedbackGenerator(style: .light)`)。
        
    - **按钮禁用逻辑**：
        
        - 未授权时，拍照按钮（Shutter）置灰且不可点击。
            
        - 点击顶部“我”、“朋友”或底部“朋友圈”入口，弹出 **Toast**：“功能开发中，敬请期待”，持续 **1.5s**，动画曲线为 `easeOut`。
            
- **限制与校验**：
    
    - **固定后置**：本阶段强制调用 `builtInWideAngleCamera`，屏蔽前置摄像头切换，符合“反自拍”定位。
        
    - **导航约束**：首页作为 `RootView`，拦截系统侧滑返回手势，防止用户回退至引导页。
        

### 3. 开发建议 (Technical Tips)

- **组件化封装**：
    
    - 建议将相机逻辑封装为 `CameraManager` 类，遵循 `ObservableObject` 协议，方便 UI 层根据 `isAuthorized` 状态实时响应。
        
    - 使用 `AVCaptureVideoPreviewLayer` 配合 `UIViewRepresentable` 引入 SwiftUI。
        
- **性能优化**：
    
    - `session.startRunning()` 是耗时操作，务必放在 **Background Thread** 执行，避免阻塞主线程导致页面卡顿。
        
- **状态持久化**：
    
    - 在 `UserDefaults` 中标记 `hasCompletedOnboarding = true`。一旦用户点击了引导页的“下一步”，后续启动 App 直接进入此首页。
- 首页默认路由：未来会做PUSH相关的逻辑，如果PUSH没有设计特定路由，或者是路由失效了，首页都是默认路由

### 4. 需求确认记录（2026-04-07）

1. **三个入口位置**：底部 Tab Bar + 顶部导航区，叠加在相机预览层上方
2. **入口视觉形态**：使用 SF Symbols 图标占位，后续替换设计稿
3. **拍照按钮位置与尺寸**：参考 iOS 系统相机拍照按钮位置，由开发自行定义尺寸
4. **拍照按钮行为**：本期仅占位，不执行拍照逻辑
5. **取消权限弹窗后状态**：保持黑屏，直到下次冷启动 App 重新触发权限检查
6. **权限弹窗弹出时机**：`denied/restricted` 状态进入页面时**立即弹出**
7. **Toast 组件**：封装为全局组件，方便后续所有页面复用

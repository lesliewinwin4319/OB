# 朋友圈页面 -- 客户端技术方案

## 1. 方案概述

这次改造把 App 的底部从原来的简单按钮升级为两个可以左右滑动切换的"页面"：左边是相机，右边是朋友圈。用户可以点底部 Tab 切换，也可以像 Snapchat 一样左右滑动整个屏幕来切换。朋友圈是一个竖向滚动的照片列表，每张照片展示"谁被拍了"、"谁拍的"、时间和点赞数。用户可以双击照片点赞（出现心形动画），单击照片进入大图浏览（支持缩放）。本期全部使用假数据，5 条 Mock 帖子，图片来自 picsum.photos。

核心技术决策：左右滑动切换用 iOS 原生的 UIPageViewController 实现（比 SwiftUI 的 DragGesture 更成熟，手势冲突更少）；大图浏览器引入 GitHub 开源库 Zoomable 实现原生相册级别的缩放体验。

## 4. UIPageViewController 包装方案

### 4.1 为什么用 UIPageViewController

SwiftUI 的 TabView(.page) 在和相机预览共存时存在手势冲突问题，而且无法精细控制过渡动画。UIPageViewController 是 UIKit 成熟方案，可以精确控制：
- 滑动方向（水平）
- 页面间的过渡样式（scroll style）
- 手势识别的优先级
- 与 ScrollView 垂直滑动的冲突隔离

### 4.2 PageContainerView 设计

```swift
struct PageContainerView: UIViewControllerRepresentable {
    @Binding var selectedTab: Int
    let pages: [AnyView]  // [透明页, FeedView]
    
    func makeUIViewController(context: Context) -> UIPageViewController
    func updateUIViewController(_ vc: UIPageViewController, context: Context)
    
    class Coordinator: NSObject, UIPageViewControllerDataSource, UIPageViewControllerDelegate {
        // dataSource: 提供前后页面
        // delegate: 滑动完成后同步 selectedTab
    }
}
```

**核心实现细节**：

1. **UIPageViewController 初始化参数**：
   - `transitionStyle: .scroll`（连续滑动，非翻页效果）
   - `navigationOrientation: .horizontal`

2. **Page 0（相机透明页）**：
   - 用一个 `UIHostingController(rootView: Color.clear)` 作为载体
   - 设置 `view.backgroundColor = .clear` 使相机透出
   - **关键**：这个页面的 view 需要 `isUserInteractionEnabled = false`，这样触摸事件能穿透到底层的 CameraView（对焦手势）
   - 但 UIPageViewController 的滑动手势识别器仍然要工作 -- 通过设置 gestureRecognizers 的 delegate 实现

3. **Page 1（朋友圈页）**：
   - 用 `UIHostingController(rootView: FeedView(...))` 包装
   - 背景色白色，完全遮挡相机

4. **selectedTab 双向同步**：
   - SwiftUI -> UIKit：`updateUIViewController` 中检测 selectedTab 变化，调用 `setViewControllers(_:direction:animated:)`
   - UIKit -> SwiftUI：`pageViewController(_:didFinishAnimating:)` 回调中更新 `selectedTab`

### 4.3 手势冲突处理

| 场景 | 水平滑动 | 垂直滑动 | 单击 | 双击 |
|------|---------|---------|------|------|
| 相机页（Page 0） | 切换到朋友圈 | 无响应 | 对焦 | 无响应 |
| 朋友圈页（Page 1） | 切回相机 | 列表滚动 | 单击照片→大图 / 单击头像→跳转 | 双击照片→点赞 |

**冲突解决策略**：

1. **相机页对焦 vs 水平滑动**：
   - UIPageViewController 的 pan gesture 自动区分水平 vs 点击
   - TapToFocusOverlay 的 `onTapGesture` 只响应点击，不和 pan 冲突
   - 无需额外处理

2. **朋友圈垂直滚动 vs 水平切换**：
   - UIPageViewController 的 scrollView 内置的 UIPanGestureRecognizer 是水平方向的
   - FeedView 的 ScrollView 的 UIPanGestureRecognizer 是垂直方向的
   - 系统自动通过方向判断来区分，但斜向滑动可能导致两者同时响应
   - 解决方案：在 PageContainerView 的 Coordinator 中，找到 UIPageViewController 内部的 scrollView，设置其 `panGestureRecognizer` 要求 `require(toFail:)` FeedView 的垂直手势。或者更简单地：给 UIPageViewController 的 scrollView 的 panGesture 设置 delegate，在 `gestureRecognizerShouldBegin` 中判断速度方向，只有水平速度 > 垂直速度时才响应

3. **单击照片 vs 双击照片**：
   - SwiftUI 原生支持：给双击手势更高优先级
   - `.onTapGesture(count: 2) { ... }` 放在 `.onTapGesture(count: 1) { ... }` 前面
   - 系统会自动等待 200ms 判断是否是双击，单击会有轻微延迟（这是预期行为）

## 5. MainTabView 详细设计

### 5.1 核心结构

```swift
struct MainTabView: View {
    @EnvironmentObject var authManager: AuthStateManager
    @StateObject private var cameraManager = CameraManager()
    @StateObject private var feedViewModel = FeedViewModel()
    @State private var selectedTab: Int = 0
    @State private var navigationPath = NavigationPath()
    
    var body: some View {
        NavigationStack(path: $navigationPath) {
            ZStack {
                // Layer 1: 相机始终在底层
                cameraLayer
                
                // Layer 2: UIPageViewController（透明页 + 朋友圈页）
                PageContainerView(
                    selectedTab: $selectedTab,
                    pages: [
                        AnyView(Color.clear),
                        AnyView(feedView)
                    ]
                )
                .ignoresSafeArea()
                
                // Layer 3: 顶部 + 底部导航栏（浮动）
                VStack {
                    TopNavigationBar(
                        isDarkMode: selectedTab == 1,
                        onTabTapped: { _ in
                            ToastManager.shared.show("功能开发中，敬请期待")
                        }
                    )
                    .padding(.top, 8)
                    
                    Spacer()
                    
                    BottomTabBar(
                        selectedTab: $selectedTab,
                        onDoubleTapFeed: {
                            // 已在朋友圈 Tab 时再次点击：滚回顶部 + 刷新
                            feedViewModel.scrollToTopAndRefresh()
                        }
                    )
                }
                
                // Layer 4: 相机权限弹窗（仅相机页）
                if selectedTab == 0 {
                    cameraPermissionAlert
                }
            }
            .navigationBarBackButtonHidden(true)
            .navigationDestination(for: PhotoFlowRoute.self) { route in
                // 拍照流程路由（保持不变）
            }
        }
    }
}
```

### 5.2 CameraView 改造要点

从当前 HomeView 中提取出纯相机相关内容：
- CameraPreviewView + TapToFocusOverlay + CaptureButton
- 移除 TopNavigationBar（上移到 MainTabView）
- 移除底部"朋友圈"按钮（被 BottomTabBar 替代）
- 移除 NavigationStack（上移到 MainTabView）
- CameraManager 的 @StateObject 上移到 MainTabView，CameraView 改为接收参数

**改造后 CameraView 职责**：仅负责渲染相机预览画面 + 拍照按钮 + 对焦动画。

注意：CameraView 不再是一个独立页面，而是 MainTabView 底层的一个组件。

### 5.3 selectedTab 与相机会话的关系

- 切换到朋友圈 Tab 时：相机 session 保持运行（不暂停），因为切回来时需要即时恢复
- 切换到朋友圈 Tab 时：CaptureButton 隐藏（通过 opacity 或条件渲染）
- 不需要额外处理，因为 UIPageViewController 的透明页会被朋友圈页覆盖

## 6. BottomTabBar 设计

### 6.1 布局

```
┌──────────────────────────────────┐
│   [相机]          [朋友圈]        │  ← 居中对齐
│    ___                           │  ← 选中指示器（黑色短横线）
└──────────────────────────────────┘
```

### 6.2 视觉规范

- 相机页背景：半透明（`Color.black.opacity(0.3)` + `.ultraThinMaterial`），让相机画面透出
- 朋友圈页背景：白色不透明
- 文字颜色：
  - 相机页：白色（选中加粗） / 白色 60% 透明度（未选中）
  - 朋友圈页：黑色（选中加粗） / 灰色（未选中）
- 选中指示器：文字下方 20pt 宽 x 2pt 高的圆角矩形，跟随切换带动画（`withAnimation(.easeInOut(duration: 0.25))`）
- 底部安全区域：自动适配（`.padding(.bottom, safeAreaInsets.bottom)`）

### 6.3 交互逻辑

```
点击 Tab：
  if 点击的 Tab == 当前 Tab:
    if 当前是朋友圈 Tab:
      → 触发 onDoubleTapFeed（滚回顶部 + 刷新）
    else:
      → 无操作
  else:
    → 切换 selectedTab（带动画）
```

### 6.4 背景色随 Tab 切换

BottomTabBar 需要接收 `selectedTab` 来决定自身背景色。切换动画期间，背景色也需要平滑过渡：

```swift
var barBackground: some View {
    Group {
        if selectedTab == 0 {
            // 相机页：半透明毛玻璃
            Rectangle()
                .fill(.ultraThinMaterial)
                .environment(\.colorScheme, .dark)
        } else {
            // 朋友圈页：白色
            Rectangle()
                .fill(Color.white)
        }
    }
}
```

## 7. FeedView + FeedViewModel 设计

### 7.1 FeedView 结构

```swift
struct FeedView: View {
    @ObservedObject var viewModel: FeedViewModel
    var onNavigateToProfile: (User) -> Void
    var onOpenImage: (Post) -> Void
    
    var body: some View {
        ScrollViewReader { proxy in
            ScrollView(.vertical, showsIndicators: true) {
                LazyVStack(spacing: 0) {
                    // 顶部留白（给 TopNavigationBar 让位）
                    Color.clear.frame(height: topInset)
                        .id("feed-top")
                    
                    ForEach(viewModel.posts) { post in
                        PostCardView(
                            post: post,
                            onTapSubject: { onNavigateToProfile(post.subject) },
                            onTapAuthor: { onNavigateToProfile(post.author) },
                            onTapImage: { onOpenImage(post) },
                            onDoubleTapImage: { viewModel.toggleLike(postId: post.id) },
                            onTapLikeButton: { viewModel.toggleLike(postId: post.id) }
                        )
                    }
                    
                    // 触发加载更多（倒数第 2 条时）
                    // 通过 .onAppear 在倒数第 2 条 PostCardView 上实现
                    
                    // 底部"到底啦"
                    if !viewModel.hasMore {
                        Text("到底啦")
                            .font(.system(size: 12))
                            .foregroundColor(Color(hex: "#999999"))
                            .frame(maxWidth: .infinity)
                            .padding(.vertical, 20)
                    }
                    
                    // 底部留白（给 BottomTabBar 让位）
                    Color.clear.frame(height: bottomInset)
                }
            }
            .refreshable {
                await viewModel.refresh()
            }
            .onChange(of: viewModel.shouldScrollToTop) { _, shouldScroll in
                if shouldScroll {
                    withAnimation {
                        proxy.scrollTo("feed-top", anchor: .top)
                    }
                    viewModel.shouldScrollToTop = false
                }
            }
        }
        .background(Color.white)
    }
}
```

### 7.2 FeedViewModel 设计

```swift
@MainActor
final class FeedViewModel: ObservableObject {
    
    @Published var posts: [Post] = []
    @Published var isRefreshing: Bool = false
    @Published var hasMore: Bool = false  // Mock 阶段固定 false
    @Published var shouldScrollToTop: Bool = false
    
    /// 加载更多闭包（预留给真实 API 接入）
    var loadMore: ((String?) -> Void)? = nil
    
    private let likeDebouncer = LikeDebouncer(interval: 0.3)
    
    init() {
        loadInitialData()
    }
    
    /// 初始加载
    private func loadInitialData() {
        posts = MockData.posts
    }
    
    /// 下拉刷新
    func refresh() async {
        isRefreshing = true
        // Mock: 延迟 1.5s 模拟网络请求
        try? await Task.sleep(nanoseconds: 1_500_000_000)
        // Mock: 重新加载数据（实际场景会从 API 拉取）
        posts = MockData.posts
        isRefreshing = false
    }
    
    /// 重击 Tab 触发的刷新
    func scrollToTopAndRefresh() {
        shouldScrollToTop = true
        Task {
            await refresh()
        }
    }
    
    /// 点赞/取消点赞（带防抖）
    func toggleLike(postId: String) {
        likeDebouncer.debounce(key: postId) { [weak self] in
            guard let self,
                  let index = self.posts.firstIndex(where: { $0.id == postId }) else { return }
            self.posts[index].isLiked.toggle()
            self.posts[index].likeCount += self.posts[index].isLiked ? 1 : -1
        }
    }
    
    /// 查询某条 post 的点赞状态（供动画组件使用）
    func isLiked(postId: String) -> Bool {
        posts.first(where: { $0.id == postId })?.isLiked ?? false
    }
}
```

### 7.3 loadMore 触发机制

在 ForEach 内部，给倒数第 2 条 PostCardView 附加 `.onAppear`：

```swift
ForEach(Array(viewModel.posts.enumerated()), id: \.element.id) { index, post in
    PostCardView(...)
        .onAppear {
            if index == viewModel.posts.count - 2 && viewModel.hasMore {
                let cursor = viewModel.posts.last?.id
                viewModel.loadMore?(cursor)
            }
        }
}
```

## 8. 数据模型

### 8.1 Post

```swift
struct Post: Identifiable {
    let id: String           // T4: String 类型（雪花 ID）
    let subject: User        // 被拍人
    let author: User         // 拍照人
    let imageURL: String     // 照片 URL
    let imageAspectRatio: CGFloat  // 图片宽高比（宽/高），用于预设占位高度
    let timestamp: Date      // 拍照人提交申请的时刻
    var isLiked: Bool        // 当前用户是否点赞
    var likeCount: Int       // 点赞总数
}
```

### 8.2 User

```swift
struct User: Identifiable, Hashable {
    let id: String
    let nickname: String
    let avatarURL: String?        // 头像 URL（本期 nil，用字母头像）
    let avatarColor: String       // 字母头像背景色 hex
}
```

注意：User 模型和现有的 Friend 模型有重叠，但语义不同（Friend 是好友关系，User 是通用用户信息），暂不合并，后续统一数据层时再考虑。

### 8.3 imageAspectRatio 说明

PRD 要求"按原图比例自适应高度"（P9）。Mock 阶段在 Post 数据中预设 imageAspectRatio，图片加载前就能计算出正确高度，避免列表跳动。真实 API 接入后，后端需返回图片宽高，客户端计算比例。

## 9. PostCardView 组件设计

### 9.1 布局结构

```
┌─────────────────────────────────────┐
│  [头像]  杰哥                        │ ← 被拍人行：头像(32pt) + 昵称
│                                     │    点击头像或昵称 → 他人页
├─────────────────────────────────────┤
│                                     │
│            [ 照 片 ]                 │ ← 宽度 = 屏幕宽 - 水平 padding
│          自适应高度                   │    单击 → 大图；双击 → 点赞动画
│                                     │
├─────────────────────────────────────┤
│  @冉冉   拍摄于2026/3/24 18:24  👍634│ ← 底部行：拍照人 + 时间 + 点赞
└─────────────────────────────────────┘
```

### 10.2 防抖设计 (LikeDebouncer)

```swift
@MainActor
final class LikeDebouncer {
    private let interval: TimeInterval
    private var pendingTasks: [String: Task<Void, Never>] = [:]
    
    init(interval: TimeInterval) {
        self.interval = interval
    }
    
    func debounce(key: String, action: @escaping @MainActor () -> Void) {
        // 取消该 key 的上一次待执行任务
        pendingTasks[key]?.cancel()
        
        pendingTasks[key] = Task {
            try? await Task.sleep(nanoseconds: UInt64(interval * 1_000_000_000))
            guard !Task.isCancelled else { return }
            action()
            pendingTasks[key] = nil
        }
    }
}
```

**注意**：防抖是在 300ms 内连续操作只执行最后一次。但 UI 层（icon 变色、数字变化）应该即时响应，不等 300ms。所以实际实现中需要考虑：

**修正方案**：UI 即时变化 + 网络请求防抖。由于本期是 Mock 无网络请求，UI 即时变化即可，防抖逻辑预留给未来 API 调用。具体做法：

```swift
struct HeartAnimationView: View {
```

### 11.3 ZoomableImageView (UIViewRepresentable)

```swift
struct ZoomableImageView: UIViewRepresentable {
    let url: String
    let cacheKey: String
    
    func makeUIView(context: Context) -> UIScrollView {
        let scrollView = UIScrollView()
        scrollView.delegate = context.coordinator
        scrollView.minimumZoomScale = 1.0
        scrollView.maximumZoomScale = 3.0
        scrollView.showsVerticalScrollIndicator = false
        scrollView.showsHorizontalScrollIndicator = false
        scrollView.bouncesZoom = true
        
        let imageView = UIImageView()
        imageView.contentMode = .scaleAspectFit
        imageView.tag = 100
        scrollView.addSubview(imageView)
        
        // 双击手势
        let doubleTap = UITapGestureRecognizer(
            target: context.coordinator,
            action: #selector(Coordinator.handleDoubleTap(_:))
        )
        doubleTap.numberOfTapsRequired = 2
        scrollView.addGestureRecognizer(doubleTap)
        
        // 单击手势（关闭大图）
        let singleTap = UITapGestureRecognizer(
            target: context.coordinator,
            action: #selector(Coordinator.handleSingleTap)
        )
        singleTap.numberOfTapsRequired = 1
        singleTap.require(toFail: doubleTap)  // 等双击判定失败后才触发单击
        scrollView.addGestureRecognizer(singleTap)
        
        // 加载图片（复用 NetworkImageView 的加载逻辑）
        // ...
        
        return scrollView
    }
    
    class Coordinator: NSObject, UIScrollViewDelegate {
        var onDismiss: (() -> Void)?
        
        func viewForZooming(in scrollView: UIScrollView) -> UIView? {
            scrollView.viewWithTag(100)
        }
        
        @objc func handleDoubleTap(_ gesture: UITapGestureRecognizer) {
            guard let scrollView = gesture.view as? UIScrollView else { return }
            if scrollView.zoomScale > 1.0 {
                scrollView.setZoomScale(1.0, animated: true)
            } else {
                let point = gesture.location(in: scrollView.viewWithTag(100))
                let zoomRect = zoomRect(for: 2.0, center: point, in: scrollView)
                scrollView.zoom(to: zoomRect, animated: true)
            }
        }
        
        @objc func handleSingleTap() {
            onDismiss?()
        }
        
        private func zoomRect(for scale: CGFloat, center: CGPoint, in scrollView: UIScrollView) -> CGRect {
            let width = scrollView.bounds.width / scale
            let height = scrollView.bounds.height / scale
            return CGRect(
                x: center.x - width / 2,
                y: center.y - height / 2,
                width: width,
                height: height
            )
        }
    }
}
```

## 17. 技术风险点

| 风险 | 说明 | 应对方案 |
|------|------|---------|
| UIPageViewController 透明页穿透触摸 | Page 0 需要 isUserInteractionEnabled=false 让触摸穿透到底层相机，但 UIPageViewController 的滑动手势也需要工作 | 只禁用 hosting VC 的 view 的交互，UIPageViewController 的 scrollView 手势独立管理 |
| 朋友圈 ScrollView 垂直滚动 vs 页面水平滑动 | 斜向滑动可能触发两个方向 | 在 UIPageViewController 的 panGesture delegate 中判断速度方向 |
| 单击 200ms 延迟 | 系统需要等双击超时才确认单击，用户感知有微小延迟 | 这是系统行为，Instagram / 微博都有同样延迟，可接受 |
| 大图浏览器双击缩放 vs 单击关闭 | 同样依赖 require(toFail:) | UIKit 手势识别器原生支持，已在方案中处理 |
| NavigationStack 上移后拍照流程兼容性 | navigationPath 从 CameraView 上移到 MainTabView | 拍照流程子页面只依赖 Binding<NavigationPath>，来源变了但接口不变 |
| picsum.photos 网络稳定性 | Mock 图片来自外部服务 | 使用 seed 确保稳定；加载失败时保持占位色，不影响其他功能 |

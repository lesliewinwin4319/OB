# 首页拍照审核流程 -- 客户端技术方案

## 1. 方案概述

用户在首页点快门后，App 用系统相机拍一张照片，同时自动存到手机相册（不管后面用户是放弃还是提交）。拍完进入确认页预览照片，确认后进入好友选择页挑选一个朋友，点击「申请」把照片发到对方主页等审核，然后自动回到首页。整个流程用 iOS 原生的 NavigationPath 做页面跳转管理，提交或放弃时一步弹回首页，不会出现页面残留。本期好友数据和上传接口全部用假数据模拟，不依赖后端。

## 3. 路由设计

### 3.1 路由枚举

```swift
enum PhotoFlowRoute: Hashable {
    case confirmation(UIImage)  // 拍照确认页，携带原图
    case uploadSelection        // 上传选择页
}
```

注意：`UIImage` 不是天然 Hashable 的，需要通过扩展或包装实现 Hashable 一致性（基于对象指针 identity）。

### 3.2 NavigationPath 状态机

```
首页 (HomeView)
  │  快门拍照成功
  │  push .confirmation(image)
  ▼
拍照确认页 (PhotoConfirmationView)
  │
  ├── 点击 [X 放弃] → 系统 Alert 二次确认
  │     ├── "是" → path = []  → 回到首页
  │     └── "否" → dismiss alert，留在当前页
  │
  └── 点击 [确定] → push .uploadSelection
        │
        ▼
      上传选择页 (UploadSelectionView)
        │
        ├── 点击 [申请] → mock API 调用 → path = [] → 回首页 + Toast
        └── 点击 [X 放弃] → Alert 二次确认 → path = [] → 回首页
```

### 3.3 NavigationPath 集成

`@State private var navigationPath = NavigationPath()` 声明在 HomeView 中，通过 `@Binding` 向下传递给子页面。子页面需要"回到首页"时，直接 `navigationPath = NavigationPath()` 清空即可。

### 3.4 侧滑返回禁用

拍照确认页和上传选择页均应用 `.disableSwipeBack()` modifier。

## 5. CameraManager 改造方案

### 当前状态
CameraManager 只管理 AVCaptureSession 生命周期，没有拍照能力。

### 改造内容
- 新增 `AVCapturePhotoOutput` 并添加到 Session
- 新增 `@Published var capturedImage: UIImage?`
- 新增 `@Published var isCapturing: Bool = false`（防快门连按）
- 新增 `capturePhoto()` 方法 + 内部 `PhotoCaptureDelegate`
- 使用 `AVCapturePhotoOutput` 而非截帧，获得完整分辨率和正确 EXIF

### capturedImage 清理时机
Pop to Root 时（navigationPath.count == 0）清 nil。

## 6. 相册保存流程 (PhotoAlbumSaver)

```
capturePhoto() 成功
    ↓ 同时并行
    ├── push 确认页（不阻塞）
    └── PhotoAlbumSaver.save(image)
            ↓
        PHPhotoLibrary.requestAuthorization(.addOnly)
            ├── .authorized/.limited → 写入相册
            ├── .denied/.restricted → Toast "请授权OB相册权限以保存照片"
            └── .notDetermined → 弹系统权限弹窗 → 同上
```

- 使用 `.addOnly` 权限，降低用户心理门槛
- 保存失败不阻塞业务流程，仅 log
- 每次拍照都触发保存（与微信一致）

## 8. 内存管理策略

| 用途 | 图片版本 | 尺寸 | 生命周期 |
|------|---------|------|---------|
| 确认页预览 | 降采样预览图 | 屏幕宽度 x 2 | 跟随 NavigationPath |
| 上传到服务端 | 原图 UIImage | 相机原始分辨率 | 跟随 NavigationPath |
| 保存到相册 | 原图 UIImage | 相机原始分辨率 | 保存完成后释放 |

Pop to Root 时，确认页和选择页 View 销毁，图片引用自动释放。

## 9. 关键交互细节

### 快门防抖
CameraManager 的 `isCapturing` 标记 + CaptureButton disabled 态

### 按钮禁用态
| 按钮 | 禁用条件 | 表现 |
|------|---------|------|
| 快门 | 未授权相机 或 拍照中 | opacity 0.3 + disabled |
| 确定 | 无 | 始终可用 |
| 申请 | 未选好友 或 提交中 | opacity 0.3 + disabled |

### 好友选择交互
| 当前状态 | 操作 | 响应 |
|---------|------|------|
| 无人选中 | 点击好友 A | A 选中，Token 出现 |
| A 已选中 | 点击好友 B | Toast "只支持选择一名好友哦" |
| A 已选中 | 再次点击 A | 无反应 |
| A 已选中 | 点击 Token X | A 移除，列表恢复 |

### 生命周期
- App 被杀：内存数据丢弃，冷启动回首页
- App 进后台：CaptureSession 自动暂停
- 来电话：通话结束保持当前页状态

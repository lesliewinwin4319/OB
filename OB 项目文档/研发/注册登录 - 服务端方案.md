# 注册登录 - 服务端技术方案

> 状态：已完成开发 & 本地验证
> 最后更新：2026-04-10

---

## 一、用一句话说清楚这个模块做了什么

用户点微信登录按钮后，服务器会认出这个微信账号是否来过，来过的直接发通行证（JWT），新来的先建账号再发通行证；填完昵称后账号变为"正常状态"，才能使用 App 的其他功能。

---

## 二、数据库设计

### users 表（核心用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | 自增整数 | 数据库内部主键，不对外暴露 |
| uid | 字符串(32) | 对外唯一标识，格式 `ob_数字`，用雪花算法生成，全局唯一 |
| openid | 字符串(64) | 微信账号标识，同一微信只对应一个 uid |
| unionid | 字符串(64) | 微信跨应用标识，预留字段 |
| nickname | 字符串(64) | 用户昵称，完善资料后写入，最多 20 字符 |
| avatar_wx_url | 长文本 | 预留：微信原始头像地址（本期不使用） |
| avatar_cdn_url | 长文本 | 预留：CDN 头像地址（本期不使用） |
| status | 枚举 | `PENDING_PROFILE`（待完善资料）/ `ACTIVE`（正常用户） |
| created_at | 时间戳 | 账号创建时间 |
| updated_at | 时间戳 | 最后更新时间 |

索引：`openid`（唯一）、`uid`（唯一）、`status`

### avatar_jobs 表（头像异步任务表，预留）

本期不使用，为未来微信头像下载至 CDN 的功能预留，表结构已建立。

## 三、接口清单

### POST /auth/wechat/login

**用途**：微信授权码换登录凭证（JWT）

**无需登录即可调用**

请求体：
```json
{ "code": "任意非空字符串（mock 模式）" }
```

成功响应（200）：
```json
{
  "accessToken": "eyJ...",
  "user": {
    "uid": "ob_300892235187093504",
    "nickname": null,
    "avatarUrl": null,
    "status": "PENDING_PROFILE",
    "createdAt": "2026-04-10T07:18:19.175Z"
  }
}
```

错误码：
- `400 BAD_REQUEST` — code 为空
- `401 UNAUTHORIZED` — 微信授权失败

---

### POST /users/me/profile

**用途**：填写昵称，完成注册（`PENDING_PROFILE` → `ACTIVE`）

**需要 JWT，`PENDING_PROFILE` 状态可调用**

请求体：
```json
{ "nickname": "测试用户" }
```

成功响应（200）：
```json
{
  "uid": "ob_300892235187093504",
  "nickname": "测试用户",
  "avatarUrl": null,
  "status": "ACTIVE",
  "createdAt": "2026-04-10T07:18:19.175Z"
}
```

错误码：
- `400 BAD_REQUEST` — 昵称为空或超过 20 字
- `401 UNAUTHORIZED` — 未登录
- `409 CONFLICT` — 资料已完善，不可重复提交

---

### GET /users/me

**用途**：获取当前登录用户信息

**需要 JWT，`PENDING_PROFILE` 状态可调用**

成功响应（200）：
```json
{
  "uid": "ob_300892235187093504",
  "nickname": "测试用户",
  "avatarUrl": null,
  "status": "ACTIVE",
  "createdAt": "2026-04-10T07:18:19.175Z"
}
```

> `avatarUrl` 字段规则：优先返回 CDN 地址，降级返回微信头像地址，均为空时返回 `null`（客户端降级为字母头像组件）

错误码：
- `401 UNAUTHORIZED` — 未登录
- `404 NOT_FOUND` — 用户不存在（理论上不触发）

---

### GET /users/me/friend-requests（预留）

**需要 JWT，必须是 `ACTIVE` 用户**

当前返回空列表：
```json
{ "items": [], "total": 0 }
```

## 四、关键设计决策

### 1. 用户状态机（最重要的安全逻辑）

```
微信登录 → PENDING_PROFILE → 填写昵称 → ACTIVE
```

- `PENDING_PROFILE` 用户只能访问白名单接口（`/users/me/profile`、`/users/me`）
- 访问其他接口返回 `403 PROFILE_INCOMPLETE`
- 状态校验以数据库实时值为准，不信任 JWT 中的状态字段（防止 token 过期前越权）

### 2. 防止同一微信账号重复创建

- 数据库层：`openid` 字段有 UNIQUE 约束，物理上不可能重复
- 应用层：先查后建（两步原子事务）+ Redis 分布式锁防并发竞争
- 效果：同一微信账号无论登录多少次，永远对应同一个 uid

### 3. UID 生成（雪花算法）

- 格式：`ob_数字`（如 `ob_300892235187093504`）
- 特点：全局唯一、有序、不暴露业务规律、未来分布式扩容兼容
- 机器 ID 当前写死为 0，扩容时按机器分配不同 ID 即可

### 4. Mock 模式（开发阶段）

- 环境变量 `WECHAT_APP_ID=wx_mock_appid` 时启用 mock
- 任意非空字符串作为 code 即可登录，无需真实微信 SDK
- 接入真实微信时：替换环境变量为真实 AppID/AppSecret，代码零修改

### 5. 本期不实现的功能（已预留）

- 头像上传（`avatar_wx_url`、`avatar_cdn_url` 字段已预留）
- 昵称修改（接口架构已支持，增加路由即可）
- 好友申请列表（接口预留，返回空列表）

## 五、错误响应统一格式

所有错误均返回：
```json
{
  "statusCode": 403,
  "errorCode": "PROFILE_INCOMPLETE",
  "message": "请先完善个人资料",
  "path": "/users/me/friend-requests",
  "timestamp": "2026-04-10T07:18:45.933Z"
}
```

## 六、本地验证结果（2026-04-10）

| 测试场景 | 预期结果 | 实际结果 |
|---------|---------|---------|
| 新用户登录 | 200，status=PENDING_PROFILE | 通过 |
| PENDING 用户访问受保护接口 | 403 PROFILE_INCOMPLETE | 通过 |
| PENDING 用户访问 /users/me | 200 正常返回 | 通过 |
| 填写昵称 | 200，status=ACTIVE | 通过 |
| 同一 code 重复登录 | 200，返回同一 uid，status=ACTIVE | 通过 |
| 重复完善资料 | 409 CONFLICT | 通过 |
| 空 code 登录 | 400 BAD_REQUEST | 通过 |
| 无 JWT 访问 | 401 UNAUTHORIZED | 通过 |
| 空昵称提交 | 400 BAD_REQUEST | 通过 |
